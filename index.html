System Architecture
01 — INTERCEPT
Transaction is caught BEFORE it's finalized on-chain. A 3-second analysis window is opened — the transaction is held in a pending state.
02 — PROFILE
Sender address is profiled — account age, transaction history, velocity (how many tx in last hour), known blacklists checked.
03 — SCORE
6 risk factors scored: velocity, amount anomaly, account age, time pattern, address reputation, transaction type risk.
04 — AI JUDGE
Claude AI reads all risk factors and writes a detailed fraud analysis report — explaining exactly why the transaction is suspicious.
05 — VERDICT
APPROVE (risk below 40%) · REVIEW (40-70%) · BLOCK (above 70%). Decision made in under 3 seconds before chain finalizes.
06 — LOG
All decisions logged immutably. Blocked transactions stored with reason. Creates audit trail for regulators and legal disputes.
